# バックエンド新機能：ユーザー詳細取得まとめ

## 1. 今日やったこと

- `GET /users/:id` エンドポイントを追加
- 認証済みユーザーのみアクセス可能

---

## 2. user.service.ts（fetchUser追加）

```typescript
export const fetchUser = async (id: number) => {
  const user = await prisma.user.findUnique({
    where: { id },
    select: {
      id: true,
      name: true,
      email: true,
    },
  });

  if (!user) {
    throw new AppError("User not found", 404);
  }

  return user;
};
```

---

## 3. user.controller.ts（getUser追加）

```typescript
export const getUser = async (req: Request, res: Response) => {
  const id = Number(req.params.id);
  const user = await fetchUser(id);
  res.status(200).json(user);
};
```

---

## 4. user.routes.ts（ルート追加）

```typescript
router.get("/:id", authMiddleware, asyncHandler(getUser));
```

---

## 5. ポイントまとめ

| 項目            | 内容                                                |
| --------------- | --------------------------------------------------- |
| `req.params.id` | URLの `:id` を取得                                  |
| `Number()`      | 文字列をnumberに変換                                |
| `findUnique`    | 1件取得・存在しない場合はnull                       |
| 404エラー       | ユーザーが見つからない場合は `AppError` を投げる    |
| 再起動          | キャッシュが残っている場合は `npm run dev` を再起動 |
